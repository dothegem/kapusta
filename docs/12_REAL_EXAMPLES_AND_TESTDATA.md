## 2. Примеры расчётов и схем

### 2.1. Примеры из Excel‑мастера

В context‑kimi приводятся выдержки из мастер‑Excel с конкретными цифрами: [file:8]  

- выручка (например, `474 500`),

Эти цифры соответствуют расчётам по схемам BELUSN/SERUSN/BELNDS/SERNDS/IPNDS и служат тест‑кейсами для автотестов калькулятора. [file:8][file:3][file:4][file:7]

В разделе 10.x context‑kimi есть набор таблиц с параметрами (часы, количество постов, длительность, итоговые суммы), показывающих различные варианты объектов охраны. [file:8]  
Каждая строка может быть напрямую интерпретирована как набор GuardCalcLine/ExpenseCalcLine и использована для проверки того, что JS‑калькулятор выдаёт те же результаты, что и Excel. [file:8]  

## 3. Использование примеров в разработке и тестировании

### 3.1. Автотесты и регрессия

Каждый реальный пример должен быть оформлен как отдельный тест‑кейс: входной state (inputs + config + calcspecVersion) и ожидаемый Output (набор Calculation с детализацией по схемам). [file:8]  
При изменении calcspec или логики калькулятора тесты должны показывать, что численные результаты совпадают с эталонными Excel‑расчётами. [file:8][file:3][file:4][file:7]  

### 3.2. Документация и обучение

Примеры можно использовать при обучении новых пользователей (менеджеров и разработчиков) для объяснения того, как устроены схемы и почему цифры получаются именно такими. [file:8]  
Наличие реальных компаний/реквизитов помогает воспринимать KAPUSTA как боевой инструмент, а не абстрактный калькулятор. [file:8]
